## Purpose

Defines the requirements for deploying the mutation test report and guardrails dashboard together to GitHub Pages on each push to main.

## Requirements

### Requirement: Mutation report is accessible at the GitHub Pages root URL
The CI pipeline SHALL ensure that the mutation report is accessible at `https://dhansak79.github.io/FocusIn/` without requiring any subpath navigation. The Pages artifact root SHALL contain an `index.html` that is the Stryker HTML report.

`htmlReporter.fileName` in `stryker.config.json` MUST be set to `"reports/mutation/index.html"` (a path relative to the working directory). This ensures the HTML report is written into the directory from which it is staged.

The `upload-pages-artifact` action SHALL upload a staging directory (`reports/pages/`) assembled before upload. The staging directory SHALL be structured as:
- `reports/pages/index.html` — the Stryker HTML report (copied from `reports/mutation/index.html`)
- `reports/pages/insights/index.html` — the guardrails impact dashboard (generated by `scripts/generate-guardrails-dashboard.js`)

A "Guardrails Dashboard" nav link SHALL be injected into the Stryker `index.html` before it is copied into the staging directory. The injection SHALL be performed by a Node.js script using `cheerio` to manipulate the DOM by targeting the `<body>` element — not by `sed` or string replacement, which would break on Stryker bundle updates.

#### Scenario: GitHub Pages root resolves to mutation report
- **WHEN** the pages workflow completes on a push to main
- **THEN** the deployed GitHub Pages site serves the Stryker HTML report at the root URL with HTTP 200

#### Scenario: Dashboard is co-deployed alongside mutation report
- **WHEN** the pages workflow completes on a push to main
- **THEN** `https://dhansak79.github.io/FocusIn/insights/` serves the guardrails impact dashboard with HTTP 200

#### Scenario: Mutation report includes link to dashboard
- **WHEN** a user visits `https://dhansak79.github.io/FocusIn/`
- **THEN** the page contains a link pointing to `/FocusIn/insights/` with accessible text "Guardrails Dashboard"

#### Scenario: Link injection survives Stryker version updates
- **WHEN** Stryker is updated and its HTML bundle structure changes
- **THEN** the cheerio injection still succeeds by targeting `<body>` rather than a specific string in the bundle
